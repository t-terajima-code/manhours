以下の仕様書に従い、勤怠データ（実働時間）と工数実績データ（総工数）を突合し、各担当者の工数が実働時間を満たしているかを検証するスクリプト(`verify_hours.py`)を作成してください。

# 1. 実行環境・制約
- **使用ライブラリ:** Pythonの標準ライブラリのみ（`csv`, `os`, `argparse`, `glob`, `re`, `html` 等）。外部ライブラリは使用禁止。
- **実行方法:** コマンドライン引数 `--month` (例: `202603` または `2026/03`) を必須とし、`--env` (デフォルト: `env`)、`--data` (デフォルト: `row_data.csv`) を受け取る。
- **対話型入力:** スクリプト実行時に `▶ 実働時間に満たない（不足している）許容割合(%)を入力してください` と標準入力で尋ね、入力された数値(float)を不足分の許容閾値とする。

# 2. ディレクトリ構成と設定ファイル (`env`)
引数で指定された `env` ファイルを読み込む。
- 1行目: ルートディレクトリ
- 3行目: `member` ディレクトリ (工数実績データ `row_data.csv` がある場所)
- 4行目: `kintai` ディレクトリ (勤怠データがある場所)
※ パス結合時は `os.path.join` のバグを防ぐため、子ディレクトリ名の先頭の `\` や `/` を `.lstrip('\\/')` で除去すること。

# 3. 勤怠データの読み込み仕様
`kintai` ディレクトリ内から、指定月を含む `.xls` ファイル（例: `*202603*kintai*.xls`）を検索する。
- **データ形式の罠対応:** 拡張子が `.xls` であっても中身がHTMLテーブルであるケースに対応するため、`<html` や `<table` が含まれる場合は正規表現（`re`）と `html.unescape` でパースすること。
- **抽出項目:** 「名前」「雇用区分」「実働時間」を取得する。
- **名前の正規化:** 名前の先頭についた社員番号（数字）や、姓名間のスペース（半角・全角）を除去したものを突合用のキーとする。

# 4. 担当者工数の集計仕様 (`row_data.csv`)
`member` ディレクトリ内の `row_data.csv` を読み込み、各担当者の総工数を集計する。
- **集計の制約条件（重要）:** 行ごとのデータのうち、**「内外製区分」と「品区コード」が両方とも空欄の行は、計算対象外として完全に除外（加算しない）**すること。それ以外の行の工数を担当者ごとに合算する。
- **休職者の除外:** 集計完了後、合計工数が完全に `0.0` の担当者は突合データから除外する。

# 5. データ突合とスキップ処理
正規化した名前を用いて、勤怠データと工数データを突合する。
- **マネージャーのスキップ:** 勤怠の「雇用区分」に「管理」または「監督」が含まれる場合、工数データが存在しなくても警告を出さずにスキップ対象とする。
- **休職・休業者のスキップ:** 勤怠の「実働時間」が `0.0` の場合、スキップ対象とする。
- **警告:** 上記以外で、片方にしか名前が存在しない場合は警告文を出力する。
- **一覧表示:** 検証実行前に、スキップした「マネージャー」と「休職・休業者等」の人数と氏名一覧をコンソールに出力する。

# 6. 検証ロジックと結果出力
- **誤差の計算式:** `誤差時間 = 工数 - 実働時間`
  （※プラスなら工数の方が多い、マイナスなら工数が不足している）
- **誤差割合の計算式:** `(誤差時間 / 実働時間) * 100` (分母が0の場合は0.0)
- **判定条件 (NGの条件):**
  工数が実働時間を下回っており（マイナス）、かつその不足割合の絶対値が、入力された「許容割合(%)」より大きい場合のみ `NG` とする。
  （※工数が実働時間を上回っている＝プラスにズレている場合は、どれだけズレていても常に `OK` とする）
- **出力フォーマット:**
  `[{OK or NG}] {名前} | 勤怠: {実働時間}h | 工数: {工数}h | 誤差: {誤差時間:+}h ({誤差割合:+}%)`
  ※誤差と割合は、プラスマイナスが明確にわかるように符号付き（`+` または `-`）で出力すること。
- **サマリ:** 最後に、NGとなった人数を出力する（0人の場合は成功メッセージを出す）。
