📋 バージョン: 2.1 📅 作成: 2026-05-26　最終更新: 2026-06-16 👤 対象者: 工数集計担当者・経理部門

はじめに

このマニュアルは、毎月の工数集計・按分処理を実行する方向けのガイドです。技術的な詳細は含まず、実行手順と確認方法に重点を置いています。

⚠ 重要： このシステムは 2段階処理 を採用しており、月初と月中旬〜下旬に分けて実行します。

第1章　システム概要（5分で理解）

システムの役割

工数集計・按分システムは、従業員が入力した工数（作業時間）を自動的に集計し、経理チームから提供される損益Excelに基づいて案件ごとにコストを計算するシステムです。

2段階処理の流れ

Stage 1　稼働5日以内
run_process.bat を実行

✅ 工数集計〜検証〜配分
✅ 工数エラーを月初に検出・修正
Stage 2　損益Excel到着後（15〜25日ごろ）
run_allocate_costs.bat を実行

✅ コスト按分（最新損益データ反映）
✅ 経理データを出力・提出
フェーズ実行時期実行ファイル処理内容所要時間

Stage 2 損益Excel到着時
（実務：15〜25日ごろ）		 run_allocate_costs.bat コスト按分
（最新損益データ反映）		 約2分

なぜ2段階か？

✅ Stage 1（稼働5日以内）：月初の早期段階で工数集計・勤怠検証・実績配分を実施。工数エラーを月初に検出し、月内に修正できます。
⏳ 損益Excel到着待ち：経理チームから損益Excelが届くまで待機（月によって15〜25日ごろ。決算スケジュール・年度末等の影響あり）。
✅ Stage 2（15〜25日ごろ）：損益Excel到着後、最新の原価データを反映したコスト按分を実施。経理システム用CSV（keiri_YYYYMM.csv）を生成します。

メリット

  1. 早期フィードバック：Stage 1 で工数エラーを月初に検出・修正
  2. 柔軟な経理連携：経理チームのスケジュール変更に対応
  3. 月内完結：稼働5日＋損益Excel到着後で月内に完了
  4. エラー削減：各段階で確認・修正が可能

第3章　事前準備（集計処理前）

Stage 1 実行前に確認するファイル

✅ 従業員の工数ファイル（member フォルダ）
  • 全従業員のCSVファイルが揃っているか
  • ファイル名形式：_W太郎.csv または 1_T太郎.csv
  • 月初までに最新データが入力されているか
✅ 勤怠データ（kintai フォルダ）
  • HTML形式の勤怠ファイルが置かれているか
  • ファイル名例：kintai_202604.html
  • 必須項目：従業員名、実働時間

Stage 2 実行前に確認するファイル

✅ 損益Excel（soneki フォルダ）
  • Excelファイル（XLSX形式）が置かれているか
  • ファイル名に事業年度番号を含む（例：事業部別損益表_189.xlsx）
  • 重要：経理チームから「損益Excel到着」通知があったか確認
  • シート名が月名（例：「5月」）になっているか

環境設定ファイル（env）の作成・確認

ファイル場所：bin\ フォルダ直下　ファイル名：env（拡張子なし）

env は、データの置き場所などをシステムに教えるための設定ファイルです。 初回や、フォルダ構成を変えたときだけ作成すればよく、毎月作り直す必要はありません。

env ファイルは 手作業で作らず、専用ツール make_env.py で作成してください。 手で作ると行のズレや空行で誤動作することがあります。ツールを使えば正しい形式で確実に作れます。

make_env.py の使い方（かんたん）

1
エクスプローラで bin\ フォルダを開く
2
アドレスバーに cmd と入力して Enter（そのフォルダでコマンド画面が開きます）
3
次のコマンドを入力して Enter
python make_env.py
4
質問が順番に表示されます。そのままで良ければ何も入力せず Enter、 変えたい項目だけ値を入力します（[ ] 内が初期値）。
プロジェクトルート(絶対パス) [C:\...\manhours]:
member サブディレクトリ [\member]:
soneki サブディレクトリ [\soneki]:
1人日の標準労働時間(時間) [7.5]:
5
最後に「env を作成しました」と表示されれば完了です。
既に env がある場合は上書きしてよいか確認されるので、よければ Y を入力します。

パスを最初から指定して作りたい場合は、引数でも渡せます（任意）：

python make_env.py --root C:\Users\t-ter\data\python\manhours

第7章　トラブルシューティング

バッチファイル（.bat）の文字化けについて： バッチファイル（run_process.bat、run_allocate_costs.bat など）は 文字化けしないよう正しい形式（日本語 Windows 用の文字コード）で保存されています。 画面のメッセージが正しく日本語で表示されます。
⚠️ 重要な運用ルール：.bat を直接エディタで編集しない

.bat ファイル（run_process.bat など）を メモ帳などのエディタで開いて直接書き換えないでください。 保存時に文字コードが変わり、画面表示が文字化けして読めなくなる原因になります。

バッチの中身を変更したいときは：

  1. システム担当者に依頼してください。
  2. 担当者は bin\make_bat.py 内の定義を編集し、python make_bat.py を実行して 正しい形式で .bat を再生成します（手編集はしません）。

このルールを守ることで、文字化けの再発を防げます。

❌ エラー1：「ファイルが見つかりません」

症状：raw_data.csv が見つかりません と表示されて止まる

原因：工数ファイルが member フォルダに入っていない、または Stage 1 が未実行

対応：

  1. member フォルダに _W従業員名.csv 形式のファイルがあるか確認
  2. Stage 1（run_process.bat）が完了しているか確認し、必要なら再実行
❌ エラー2：「損益Excelファイルが読めません」

症状：Excelファイル 'xxx.xlsx' が読み込めませんでした

原因：損益ファイルが見つからない、または形式が違う

対応：

  1. soneki フォルダに .xlsx ファイルがあるか確認（.xls や .xlsm は不可）
  2. Excelで開いて、シート名が「4月」「5月」など月名になっているか確認
  3. 経理部門に最新ファイルを再送付依頼
❌ エラー3：出力ファイルが空または 0 になっている

症状：raw_data.csv が生成されたが全部 0

原因：従業員の工数ファイルが見つからない、または形式が違う

対応：

  1. member フォルダ内のCSVファイルをExcelで開いて形式を確認
  2. ファイル名が _W で始まるか、または 数字_T で始まるか確認
  3. 形式が違う場合は担当者に修正を依頼
❌ エラー4：「環境設定ファイルが見つかりません」

症状：設定ファイル 'env' が見つかりません

原因：env ファイルがない、またはパスが違う

対応：

  1. bin\ フォルダに env（拡張子なし）があるか確認
  2. ない場合は bin\ フォルダで python make_env.py を実行して作成（第3章「環境設定ファイル（env）の作成・確認」を参照）
❌ エラー5：「'python' は内部コマンドとして認識されていません」

症状：バッチファイル起動直後に上記メッセージが出て終了する

原因：PCに python コマンドがPATH登録されていない（py ランチャーのみの環境）

対応：

  • バッチファイルは python が見つからない場合に 自動的に py コマンドで再試行 します。通常はそのまま処理が続きます。
  • それでも失敗する場合は、Python公式サイトから Python をインストールし、「Add Python to PATH」にチェックを入れて再インストールしてください。
⚠️ 警告：「工数の合計が実働時間と異なります」

症状：verify 結果に警告が記録されている

原因：従業員の入力工数と勤怠システムの実働時間に差がある

対応：

  • 軽度の差異（±1時間以内）：許容範囲 → そのまま進める
  • 大きな差異（±5時間以上）：当人に確認 → 修正後に再実行
  • 判断できない場合は管理者に報告

付録　FAQ（よくある質問）

Q1: Stage 1 と Stage 2 をまとめて実行できませんか？
いいえ。損益Excel待ちの期間があるため、2段階に分ける必要があります。
Q2: 損益Excelがいつ来るかわかりません
経理チームに「損益Excel予定日」を事前に確認することをお勧めします。
Q3: 毎月同じ手順ですか？
はい。月号を変えるだけで、手順は同じです。
Q4: Stage 1 で工数エラーが見つかりました。どうしましょう？
入力者に修正を依頼します。Stage 1 を月初に実行することで、月内に対応できます。
Q5: 複数月をまとめて処理できますか？
1ヶ月ずつの実行を推奨します。
Q6: データのバックアップは？
results/ と keiri/ フォルダを定期的にコピーしておくことを推奨します。
